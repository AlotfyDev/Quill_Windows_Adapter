# Impact Analysis: LOGV Variable-Name Macros

## Summary
Total GAPs: 3 | P0: 1 | P1: 1 | P2: 1 | API Changes: 1

## GAP Details
| Gap ID | Severity | Issue | Production Impact | API Change? | Decision |
|--------|----------|-------|-------------------|-------------|----------|
| GAP-4-M03-1 | 🔴 P0 | AA spec example `LOGV_INFO(order_log, "order_id", order_id, "price", price)` passes string literals. Quill's `QUILL_GENERATE_FORMAT_STRING` uses `#x` preprocessor stringification which wraps bare identifiers — string literals produce `"price"` (with quotes in key name). Correct call uses bare identifiers: `LOGV_INFO(order_log, "trade", order_id, price)`. | Operators following the AA spec will produce output with quoted key names like `order_id:12345` instead of `order_id:12345`. Log parsing tools searching for `order_id=` will miss entries. Structured debugging broken. | Yes — macro usage convention changes (not macro signature, but caller convention) | 🛠️ Fix now — Fix the example, add comment explaining `#var` stringification. |
| GAP-4-M03-2 | ⚠️ P1 | AA spec expected output `[INFO] order_id=12345, price=150.25` does not match Quill v10.0.1 format. Actual Quill format is `text [identifier: value, ...]` as generated by `QUILL_GENERATE_FORMAT_STRING`. So correct output would be: `trade [order_id:12345, price:150.25]` after the default pattern prefix. | Acceptance tests will falsely fail (or be written to match wrong format). Operators relying on the AA doc for log parsing will write broken parsers. | No | 🛠️ Fix now — Update expected output to match real Quill format. Document `%(named_args)` behavior. |
| GAP-4-M03-3 | 🟡 P2 | AA spec does not define `LOGV_BACKTRACE`, `LOGV_LIMIT`, or `LOGV_LIMIT_EVERY_N` wrappers — but Quill provides these macro families (e.g., `LOGV_INFO_LIMIT`, `LOGV_CRITICAL_LIMIT_EVERY_N`). Users needing rate-limited structured logging must drop to raw Quill. | Power users bypass Logger_Adapter for these features, reducing the adapter's value. Not a correctness issue. | Yes — if added | ❌ Accept — Document this as future work in the AA spec. Add a note that Quill's `LOGV_*_LIMIT` macros can be used directly. |

## Recommended AA Changes
- **GAP-4-M03-1**: Replace `LOGV_INFO(order_log, "order_id", order_id, "price", price)` with `LOGV_INFO(order_log, "trade", order_id, price)`. Add explanatory comment: "// LOGV uses #var preprocessor stringification — pass bare identifiers, not string literals."
- **GAP-4-M03-2**: Replace expected output `[INFO] order_id=12345, price=150.25` with correct format. Add note: "Quill v10.0.1 generates format: `description [var_name: value, ...]`"
- **GAP-4-M03-3**: Add "Future Work" note: `LOGV_LIMIT`, `LOGV_BACKTRACE`, `LOGV_TAGS` wrappers are available directly from Quill — document that advanced users can use `QUILL_LOGV_INFO_LIMIT(...)` or the non-prefixed `LOGV_INFO_LIMIT(...)` if `QUILL_DISABLE_NON_PREFIXED_MACROS` is not defined.
